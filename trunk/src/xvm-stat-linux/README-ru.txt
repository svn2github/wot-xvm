XvmFS - это виртуальная файловая система на основе FUSE. Используется для 
отображения статистики игроков в моде XVM.

XVM - популярный мод для многопользовательской онлайновой игры 'World of Tanks'.

XVM, XVM-STAT и XvmFS распространяются по свободной лицензии GNU GPL v3.
Исходный код и готовый мод доступен по адресу http://code.google.com/p/wot-xvm/
Установка и настройка XVM подробно описывается на официальном сайте проекта.
Подразумевается, что XVM уже установлен и настроен.


Для работы XvmFS требуется: 
 - Поддержка FUSE со стороны ядра. Работает на ядрах 2.4.X и 2.6.X, включено 
   по умолчанию в большинстве современных дистрибутивов.

 - Интерпретатор Python. Также включен в базовую версию большинства дистрибутивов.
   При разработке использовалась версия питона 2.6.5

 - Библиотеки FUSE для Python. Нужно установить вручную.
   Под Ubuntu устанавливается так:

   $ sudo apt-get install python-fuse


XvmFS должен быть смонтирован в каталог res_mods/.stat , которая находится в
основном каталоге игры. Если этого каталога не существует, его необходимо
создать:

$ cd ~/.wine/drive_c/Games/World_of_Tanks
$ mkdir -p res_mods/.stat

Есть один нюанс: XvmFS не может автоматически определить, как будет выглядеть
путь к папке с игрой для самой игры, запущенной под WINE. С другой стороны,
XvmFS должен передать это значение по запросу в XVM. Поэтому у XvmFS появилась
опция '-o game_dir=...', которую надо использовать в обязательном порядке.

То же, но короче: ВСЕГДА запускайте XvmFS с опцией '-o game_dir=...', указывая
в ней такой путь, каким он должен выглядеть для игры, запущенной под WINE.
Например, если игра установлена в ~/.wine/drive_c/Games/World_of_Tanks :

$ ./xvmfs res_mods/.stat -o game_dir='C:\Games\World_of_Tanks'

Опция '-o log_dir=...' указывает, в каком каталоге создавать лог-файлы.
По умолчанию логи пишутся в текущий каталог.

Для работы XvmFS требуются два файла с настройками:
 - Настройки лаунчера World of Tanks - файл называется 'WOTLauncher.cfg' и
   находится в основной папке с игрой.

 - Настройки XVM-STAT - файл 'xvm-stat.exe.config' от базовой версии xvm-stat,
   распространяется вместе с XvmFS

По умолчанию считается, что оба два файла находятся в текущем каталоге. Если это
не так, то местонахождение этих файлов можно указать опциями '-o wot_cfg=...' 
и '-o xvm_cfg=...' соответственно.

Если статисика в игре не отображается, в логах должны быть сообщения о том, что
и как происходит. Если в логах попадаются сообщения с 'HTTP Error' - то
скорее всего, проблема в большой нагрузке на внешний прокси-сервер, к которому
идёт обращение из XvmFS. Самостоятельно исправить это никак не получится.

Для более подробного логирования XvmFS можно запустить с опцией '-o debug'.

Перед первым запуском убедитесь, что основной скрипт xvmfs доступен к исполнению.
Установить права на исполнение файла можно так:

$ chmod +x xvmfs

После того, как XvmFS смонтирована в нужный каталог, можно запускать игру точно
так же, как это делалось ранее. При перезапуске игры дополнительно делать ничего не надо.
Рекомендуется размонтировать XvmFS после того, как Вы закончили играть:

$ fusermount -u res_mods/.stat
